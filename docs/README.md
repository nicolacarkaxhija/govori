# Documentation

Documentation follows a **registry pattern**: every subdirectory has a
`README.md` index listing its documents with one-line summaries. Read the
registry first; open only the files you need.

| Registry                          | Contents                                                                  |
| --------------------------------- | ------------------------------------------------------------------------- |
| [adr/](adr/README.md)             | Architecture Decision Records — every significant decision with rationale |
| [prd/](prd/README.md)             | Product requirement documents — one concern per file                      |
| [tech/](tech/README.md)           | Technical specifications — one subsystem per file                         |
| [process/](process/README.md)     | Working agreements: contribution, licensing, estimates, governance        |
| [marketing/](marketing/README.md) | Staged outreach and growth plan                                           |
